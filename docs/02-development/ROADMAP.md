# Дорожная карта разработки сайта "Halcyon"

Этот документ описывает пошаговый план разработки сайта с использованием Astro. Каждый этап разбит на небольшие, конкретные задачи, предназначенные для выполнения ИИ-ассистентом.

## Этап 0: Настройка и Основы (Setup & Foundation)

- [x] **0.1. Инициализация проекта:** Настроить новый проект Astro с базовой конфигурацией (TypeScript, Prettier).
- [x] **0.2. Структура папок:** Создать стандартную для Astro структуру директорий:
  - `src/components/` (для переиспользуемых UI-компонентов)
  - `src/layouts/` (для макетов страниц)
  - `src/assets/` (для изображений, иконок, шрифтов)
  - `src/content/` (для хранения контента в Markdown файлах)
  - `src/styles/` (для глобальных стилей)
- [x] **0.3. Глобальные стили:** Создать файл `src/styles/global.css` и определить в нем основные переменные (цвета, шрифты) согласно ТЗ.
- [x] **0.4. Базовый макет:** Создать основной макет сайта `src/layouts/BaseLayout.astro`, включающий в себя шапку (Header) и подвал (Footer).

## Этап 1: Компоненты и Главная страница

- [x] **1.1. Компонент Header:** Создать компонент `src/components/Header.astro` с логотипом, навигацией и переключателем языков.
- [x] **1.2. Компонент Footer:** Создать компонент `src/components/Footer.astro` с контактной информацией, ссылками на соцсети и копирайтом.
- [x] **1.3. Компонент Hero:** Создать компонент для первого экрана главной страницы с заголовком, подзаголовком и кнопкой "Получить консультацию".
- [x] **1.4. Компонент преимуществ:** Создать компонент для отображения ключевых преимуществ компании.
- [x] **1.5. Сборка Главной страницы:** Собрать главную страницу `src/pages/index.astro`, используя созданные компоненты.

## Этап 2: Контент и Внутренние страницы

- [x] **2.1. Страница "О компании":** Создать страницу `src/pages/about.astro`. Контент для нее будет храниться в Markdown.
- [x] **2.2. Страница "Контакты":** Создать страницу `src/pages/contact.astro` с адресом, телефонами и формой обратной связи.
- [x] **2.3. Настройка коллекций контента (Astro Content Collections):**
  - Настроить коллекцию для "Услуг" в `src/content/config.ts`.
  - Настроить коллекцию для "Блога/Публикаций".
- [x] **2.4. Создание страниц услуг:** Создать динамический роут `src/pages/services/[slug].astro`, который будет генерировать страницы для каждой услуги из Markdown-файлов.
- [x] **2.5. Создание общей страницы услуг:** Создать страницу `src/pages/services/index.astro`, которая будет выводить список всех услуг.
