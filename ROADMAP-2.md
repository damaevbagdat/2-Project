# ROADMAP-2: Подготовка к запуску (Production)

Техническая разработка по основной дорожной карте завершена. Этот документ описывает следующие шаги для подготовки сайта к запуску на реальном домене `alash-zan.com`.

## 1. Наполнение реальным контентом

- [ ] **Иконки (Favicons):** Заменить все placeholder-файлы в папке `public/` (например, `favicon.ico`, `apple-touch-icon.png`) на настоящие иконки бренда.
- [ ] **Тексты и изображения:** Проверить и заменить все временные тексты и изображения (например, в `Hero.astro` и во всех `.md` файлах в `src/content/`) на финальные, предоставленные юристами.

## 2. Конфигурация для домена `alash-zan.com`

- [ ] **Настройка домена в GitHub:** В настройках репозитория, в разделе "Pages", указать кастомный домен `alash-zan.com`.
- [ ] **Настройка DNS:** У регистратора доменов (где был куплен `alash-zan.com`) нужно будет прописать CNAME или A-записи, указывающие на серверы GitHub Pages.
- [x] **Обновление `astro.config.mjs`:** Изменить `site` и `base` обратно на основной домен.

## 3. Настройка рабочих сервисов

- [ ] **Аналитика:** В файле `src/layouts/BaseLayout.astro` заменить placeholder ID `G-XXXXXXXXXX` и `XXXXXXX` на реальные ID из Google Analytics и Яндекс.Метрики.
- [ ] **Форма обратной связи:** Чтобы форма отправляла уведомления на почту или в Telegram, необходимо в настройках репозитория на GitHub (в разделе `Settings -> Secrets and variables -> Actions`) добавить соответствующие секреты (`SMTP_HOST`, `TELEGRAM_BOT_TOKEN` и т.д.), как описано в `PR_FULL.md`.

## 4. Финальное тестирование и запуск

- [ ] **Финальное тестирование:** После всех настроек провести полное тестирование сайта на реальном домене (проверить все ссылки, формы, языковые версии).
- [ ] **Запуск:** Официально анонсировать запуск сайта.
